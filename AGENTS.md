# AGENTS.md — operating guide for scheduled / future runs

This file is the **durable handoff**. The pipeline is designed so that **all state lives
in git + GitHub Issues + the vault token** — there is NO hidden state in any agent
sandbox. To resume (e.g. a daily scheduled run), an agent needs only: (a) these two
repos, (b) the GitHub PAT, (c) this file.

## Mission
Track **benchmark citations** across the LLM ecosystem. Topline entity = the **benchmark**
(not the model). For each benchmark: how often cited, doc_type, prominence, citing
lab/model, methodology deviations, and whether it's already in Harbor.

## Where state lives (durable, no sandbox dependency)
- `christicode/benchmark-citations` — pipeline + flat-file data (this repo).
  - `labs.yaml` — in-scope labs as CONFIG (+ `source_index_urls` to crawl). Onboard a lab = edit this only.
  - `data/citations.jsonl` — the citation graph (generated by `scripts/build_citations.py`).
  - `data/aliases.yaml` (+ `aliases_generated.yaml`) — deterministic normalization registry.
  - `schema/extraction_schema.json` — citation record shape.
  - `scripts/extract.py` — **discovery**: doc → detected benchmarks → normalized (flags unmatched).
  - `scripts/build_citations.py` — curated backfill dataset (embeds human-verified extraction).
  - `scripts/rank.py` — conversion-candidate ranking (live Harbor cross-ref).
  - `scripts/review_queue.py` — live view of open `needs-human-review` issues.
- `christicode/benchmark-citations-sources` — raw source PDFs (Git LFS) + `manifest.yaml`
  (source_url + sha256 per doc). Rehydrate with `scripts/fetch_sources.py`.
- **GitHub Issues** (label `needs-human-review`) — the review queue's single source of truth.
- **Harbor**: `harbor-framework/harbor` `registry.json`, fetched **live from main** each run.
  Dedupe by `(name, version)` — names are NOT unique.

## Secrets / permissions (in the vault)
GitHub fine-grained PAT on `christicode` with **Administration + Contents + Workflows +
Issues: write**, All repositories. Network access for crawling + `curl` PDF fetch
(`curl` succeeds on CDNs where the hosted web-fetch is blocked). `pdftotext` (poppler) for text.

## Daily scheduled run — procedure (feeds → candidates → human promotes → extract)
1. **Discover.** For each lab in `labs.yaml`, crawl its `source_index_urls`; list model
   cards / system cards / headline blogs published since the last run. **Enumerate from
   the index — never from a guessed list of model names** (that's how Mythos/Fable was
   first missed; discover by crawling, then diff).
2. **Diff.** Drop docs already in `data/citations.jsonl` (`source_doc.url`) and
   `sources/manifest.yaml`. What remains = new candidates.
3. **Present candidates for HUMAN promotion.** Do NOT auto-extract sight-unseen. Open (or
   update) a candidates issue listing new docs. Wait for human promotion.
4. **On promoted docs:** archive the raw PDF to `benchmark-citations-sources` (append to
   `manifest.yaml` with sha256); run `scripts/extract.py` to discover benchmarks; read
   exact scores from tables (curated, reviewed); add records to `data/citations.jsonl`.
   Any unmatched name → open a `needs-human-review` issue; never guess.
5. **Rank.** Run `scripts/rank.py` (live Harbor cross-ref) → refresh conversion candidates.
6. **Dashboard.** Regenerate `docs/` (GitHub Pages) — rising usage, rising saturation, top
   candidates + one-line "why now". Commit.

## Guardrails (hard rules)
- **Never** `git push`, create a repo, open/close issues, or publish the dashboard without
  **explicit human confirmation**, stating exactly what will be published. Local commits are free.
- **Never fabricate** a citation or methodology detail. If a doc is ambiguous or a card is
  image-only, **flag for review** (GitHub issue) — don't guess.

## Known gotchas
- Some cards present benchmarks as **images only** (Mistral Large 3, Gemini 3 Pro model
  card) → numbers must come from OCR or the blog; flag, don't invent.
- **Config-dependent scores**: Fable 5 reroutes cyber/bio to Opus 4.8 (0.0%); headline
  numbers are the unblocked Mythos 5 — always record `model_config`.
- **Harness matters**: Terminal-Bench via Terminus 1 vs Terminus 2 vs Claude Code gives
  different scores — capture it.
- **HLE / SWE-bench** etc. get restated after grader/scaffold changes — a sha256 mismatch in
  `manifest.yaml` means the upstream doc changed; re-review.
- **Large/binary files can't go through the MCP file API** — use the `regenerate` and
  `archive-sources` Actions (server-side) or a local `git`/`git-lfs` push.
